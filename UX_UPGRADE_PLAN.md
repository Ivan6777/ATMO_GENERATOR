# План апгрейду UI/UX — Platform Atmo (Studio Pro)

> Мета: зробити редактор максимально **дружнім** (новачок робить перше відео без інструкції)
> та **зручним** (досвідчений користувач працює швидко, без зайвих кліків).
> План складено за аудитом поточного коду `VideoApp.jsx` (Synthesia-layout: сцени → канвас → властивості → таймлайн).

## 1. Аудит поточного стану

### Сильні сторони (зберегти)
- Впізнаваний layout відеоредактора: зліва сцени, по центру канвас, справа властивості, знизу таймлайн.
- Уже є: Undo/Redo (Ctrl+Z/Y), вставка з PowerPoint (Ctrl+V), панель шарів, зум таймлайну,
  drag&drop анімацій на таймлайні, тримінг медіа, AI-режисер (озвучка + анімації + переходи), журнал змін.
- Єдиний акцентний колір `#7c3aed` і послідовна стилістика Tailwind-класів.

### Болючі точки (виправляє цей план)

## 2. Принципи дизайну (для всіх фаз)

1. **Прямі маніпуляції**: редагувати там, де бачиш (канвас), панель — лише для тонких налаштувань.
2. **Видимий стан системи**: кожна довга операція має прогрес, кожна тиха дія — тост із «Скасувати».
3. **Прощення помилок**: усе відворотне; деструктивні дії — з м'яким підтвердженням (undo-снекбар, не модал).
4. **Прогресивне розкриття**: спершу 3 головні дії (Імпорт → Озвучити → Експорт), деталі — за запитом.
5. **Швидкість для профі**: все, що є мишею, має бути і з клавіатури.

## 3. Фази впровадження

### Фаза 0 — Швидкі перемоги (≈1–2 дні, максимальний ефект/зусилля)

| Задача | Деталі реалізації | Критерій готовності |
|--------|-------------------|---------------------|
| 0.1 Автозбереження | Серіалізувати `videoState.slides` у IndexedDB (localStorage замалий для base64-аудіо) з дебаунсом 2с; банер «Відновити останню сесію?» на старті | Закрив вкладку → відкрив → проєкт на місці |
| 0.2 Undo-снекбар | Після видалення слайда/об'єкта — тост «Слайд 3 видалено · Повернути (Ctrl+Z)» 5с | Видалення без страху |
| 0.3 Модал гарячих клавіш | Кнопка «?» у топбарі + клавіша `?`: список усіх шорткатів (згенерувати з keydown-handler) | Всі шорткати видно в UI |
| 0.4 Тост-система | Єдиний компонент `Toast` (success/error/info + дія); підключити до `logChange` для ключових подій (експорт готовий, озвучка готова, помилка API) | Жодної «тихої» важливої події |
| 0.5 Прогрес імпорту PPTX | `extractPptxData` вже йде по слайдах циклом — прокидати callback `onProgress(i, total)`; оверлей «Розбираємо слайд 4/12…» | Імпорт великого файлу не виглядає зависанням |
| 0.6 Прогрес озвучки всіх слайдів | У `generateAllMissingSlidesAudio` — список слайдів зі статусами (⏳/✅/❌ + retry окремого) | Видно, який слайд озвучується і де помилка |
| 0.7 Індикатор незбережених змін | Крапка на назві проєкту + `beforeunload`-попередження, якщо є зміни після останнього експорту/збереження | Випадкове закриття не втрачає роботу |

### Фаза 1 — Онбординг і перший успіх (≈2–3 дні)

| Задача | Деталі |
|--------|--------|
| 1.1 Екран завантаження 2.0 | Велика drag&drop зона (вже є `handleDrop`) + 3 картки: «Відкрити PPTX», «Продовжити з MP4», «Почати з порожнього» + міні-скріншоти можливостей |
| 1.2 Демо-режим | Кнопка «Спробувати на демо» — вшита маленька презентація (2–3 слайди з формулою, відео, тестом) для гри без свого файлу |
| 1.3 Інтерактивний тур | 4 кроки-спотлайти при першому вході: ① сцени ② канвас (подвійний клік = редагувати) ③ озвучка ④ експорт; «Не показувати знову» у localStorage |
| 1.4 Порожні стани | Порожній слайд: по центру канваса — привид-підказка «Додайте текст, зображення або віджет» з кнопками; порожня панель властивостей: «Виділіть об'єкт на слайді» |
| 1.5 Чекліст готовності до експорту | Біля кнопки «Експорт»: ✅ слайдів N · ✅/⚠️ озвучено M з N · підказка «2 слайди без озвучки — згенерувати?» |

### Фаза 2 — Редактор канваса: прямі маніпуляції (≈4–5 днів, ядро зручності)

| Задача | Деталі |
|--------|--------|
| 2.1 Інлайн-редагування тексту | Подвійний клік по текст-об'єкту → contentEditable поверх канваса зі стилями рядків; Esc/клік поза — зберегти. Math-руни показувати як «чіпи» (não редаговані, зберігаються як є). Це прибирає розрив Б1 |
| 2.2 Smart guides + прилипання | При drag/resize: пунктирні напрямні до центру слайда, країв, центрів/країв сусідніх об'єктів; прилипання ±4px; Alt = тимчасово вимкнути |
| 2.3 Мультивиділення | Рамка виділення по порожньому місцю, Shift+клік — додати; групові move/delete/align; панель «Вирівняти: ліво/центр/право/верх/низ + розподілити» |
| 2.4 Контекстне меню | ПКМ по об'єкту: Дублювати (Ctrl+D), Копіювати стиль, На передній/задній план, Видалити; ПКМ по канвасу: Вставити, Змінити фон |

### Фаза 3 — Таймлайн і озвучка (≈3–4 дні)

| Задача | Деталі |
|--------|--------|
| 3.1 Waveform озвучки | Малювати хвилю PCM (`audioBase64` вже є) на треку слайда — видно паузи й межі речень |
| 3.2 Прев'ю голосу | Кнопка ▶ біля вибору голосу: озвучити фіксовану фразу цим голосом (кеш зразків); групи голосів як картки зі статтю/прапорцем, а не довгий select |
| 3.3 Караоке-підсвітка | Під час прев'ю підсвічувати поточне речення в тексті диктора (оцінка ~2.5 слова/с вже використовується в AI-промпті) |

## 4. Дизайн-система (наскрізна, впроваджується з Фази 0)

**Токени** (винести в константи/CSS-змінні):
- Кольори: `primary #7c3aed`, `primary-hover #6d28d9`, success `emerald-500`, danger `red-500`,
  поверхні `white/slate-50/slate-100`, текст `slate-800/500`, межі `slate-200`.
- Відступи: шкала 4px (4/8/12/16/24); радіуси: 8 (кнопки/поля), 12 (картки/модали); тіні: 2 рівні.
- Типографіка: 11px (підписи) / 12–13px (контроли) / 14px (тексти) / 16–20px (заголовки).

**Базові компоненти** (зараз стилі дублюються інлайном ~150 разів):
`<Btn variant>`, `<IconBtn>`, `<Panel title collapsible>`, `<Field label>`, `<Tooltip>`, `<Modal>`, `<Toast>`, `<Kbd>`.
Кожна нова фаза використовує лише їх — старі місця мігрують поступово.

## 5. Метрики успіху

| Метрика | Зараз (оцінка) | Ціль |
|---------|----------------|------|
| Time-to-first-video (новачок, демо) | невимірно / >15 хв | ≤ 5 хв |
| Кліків від імпорту до експорту (без правок) | ~8–10 | ≤ 4 |
| Втрата роботи при закритті вкладки | 100% | 0% (автозбереження) |
| Редагування тексту: дій до зміни одного слова | 4 (виділити → панель → знайти рядок → правка) | 2 (подвійний клік → правка) |
| Icon-кнопок без aria-label | ~150 | 0 |
| Фриз UI при імпорті 40-слайдового PPTX | секунди | немає (прогрес + yield) |

Заміри: панель `Logger` вже фіксує події — додати тайм-стемпи сесії та лічильник кліків до першого експорту.

## 7. Ризики та як їх знімаємо

- **Моноліт 8k рядків**: кожну фазу супроводжує м'яка модуляризація (нові компоненти — окремими блоками файлу з чіткими секціями), без big-bang рефакторингу.
- **Регресії канваса**: перед Фазою 2 — зафіксувати еталонні знімки `drawSlideFrame` (уже є тестова інфраструктура jsdom+esbuild зі скретчпаду) для 5–6 еталонних слайдів.
- **babel-standalone сумісність**: не додавати залежності, які потребують збирача; все — vanilla React + інлайн SVG (як зараз).
- **Touch/planshет**: перевірити pointer events на реальному пристрої до релізу Фази 4.
